# Contributing to Anamnesis

Anamnesis uses an upstream-first contribution model.

## Before you start

- Read [`LICENSE`](LICENSE). This repository is source-available and
  noncommercial, not open source.
- If you maintain a fork with modifications, send those changes back to the
  canonical repository as a pull request, patch series, or another format the
  maintainers accept.
- If you want to discuss commercial use, contact the maintainers before using
  the code in a paid product or service.

## Contribution expectations

- Keep changes focused and reviewable.
- Include tests or a clear explanation for why tests are not practical.
- Preserve attribution, copyright, warranty, and provenance notices.
- Mark substantial user-visible behavior changes clearly in the PR description.

## Inbound contribution terms

Unless separately agreed in writing, any contribution intentionally submitted
for inclusion in Anamnesis is provided under the repository license and also
includes permission for the Anamnesis maintainers to use, reproduce, modify,
distribute, sublicense, and relicense that contribution as part of Anamnesis,
including for commercial licensing by the maintainers.

Only submit material that you have the right to contribute.
